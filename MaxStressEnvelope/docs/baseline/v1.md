# MaxStressEnvelope / baseline / v1

## 当前实现

baseline 是多工况最大应力包络的单线程参考实现。输入数据采用 element-major 布局：

```text
idx = i * N + j
```

其中 `i` 是单元编号，`j` 是工况编号，`N` 是工况数。每个单元连续存放它在所有工况下的六个应力分量。

核心计算是：对每个单元遍历所有工况，计算当前工况的 Von Mises 应力，并保留最大值。

```cpp
for (int i = 0; i < M; ++i) {
    float max_vm = 0.0f;
    for (int j = 0; j < N; ++j) {
        int idx = i * N + j;
        float ds = sx[idx] - sy[idx];
        float ss = sy[idx] - sz[idx];
        float ts = sz[idx] - sx[idx];
        float vm = std::sqrt(0.5f * (ds*ds + ss*ss + ts*ts +
                                     6.0f * (txy[idx]*txy[idx] + tyz[idx]*tyz[idx] + tzx[idx]*tzx[idx])));
        max_vm = std::max(max_vm, vm);
    }
    envelope[i] = max_vm;
}
```

源码位置：`MaxStressEnvelope/kernels/baseline.cpp`

## 优化程度

这是 v1 的正确性和性能基线。它不使用线程并行、显式 SIMD 或 GPU，只保留最直接的双层循环。

| 方向 | 当前状态 |
|---|---|
| 线程级并行 | 无，单线程 |
| 工况归约 | 单元内串行 max |
| 数据布局 | element-major，每个单元的工况连续 |
| SIMD | 不显式使用 |
| GPU | 不使用 |

## 性能特征

这个算子比单次 Von Mises 多一层工况归约。每个输出 `envelope[i]` 需要读取 `N` 个工况的六个应力分量，并完成 `N` 次 Von Mises 计算和最大值比较。

耗时主要来自：

- `M * N` 规模的连续输入读取。
- 每个工况一次平方根计算。
- 每个单元内部的 `max_vm` 归约依赖。
- 单线程无法利用多核或 GPU 并行。

## 为什么需要 baseline

baseline 让“多工况包络”的计算语义最清楚：先算每个工况的等效应力，再取单元级最大值。OpenMP、SIMD、CUDA 和 CUB 都是在这个语义上改变并行层次或归约实现。
