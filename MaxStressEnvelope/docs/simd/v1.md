# MaxStressEnvelope / SIMD / v1

## 当前实现

SIMD v1 保持 element-major 数据布局和双层循环结构。内层工况循环连续访问同一单元的所有工况：

```cpp
for (int i = 0; i < M; ++i) {
    float max_vm = 0.0f;
    for (int j = 0; j < N; ++j) {
        int idx = i * N + j;
        float ds = sx[idx] - sy[idx];
        float ss = sy[idx] - sz[idx];
        float ts = sz[idx] - sx[idx];
        float vm = std::sqrt(0.5f * (ds*ds + ss*ss + ts*ts +
                                     6.0f * (txy[idx]*txy[idx] + tyz[idx]*tyz[idx] + tzx[idx]*tzx[idx])));
        max_vm = std::max(max_vm, vm);
    }
    envelope[i] = max_vm;
}
```

源码位置：`MaxStressEnvelope/kernels/simd.cpp`

## 优化程度

这是 v1 的自动向量化实现。当前没有手写 AVX/SSE intrinsic，也没有把工况归约拆成专门的向量归约代码，主要依赖 Release 编译参数：

- `-O3`
- `-march=native`
- `-ffast-math`

| 方向 | 当前状态 |
|---|---|
| SIMD 方式 | 编译器自动向量化尝试 |
| 显式 intrinsic | 无 |
| 数据布局 | 每个单元的工况连续 |
| 归约 | `max_vm` 单元内最大值 |
| 适合目标 | 观察连续内层循环能被编译器优化到什么程度 |

## 优化思路

相比 CSR-SpMV，这个算子的内层循环更适合 SIMD：

- `sx[idx]` 到 `tzx[idx]` 都是连续访问。
- 没有 `x[col_idx[k]]` 这类间接访存。
- 没有分支。
- 每个工况的 Von Mises 计算结构相同。

编译器可以尝试对多个工况同时计算 Von Mises，并把多个 `vm` 值做向量化 max 归约。

## 性能限制

当前 v1 仍有几个限制：

- `max_vm` 是归约变量，编译器是否向量化取决于优化能力和 `fast-math` 条件。
- `sqrt` 的向量化成本较高。
- `N` 如果很小，向量 lanes 可能不容易填满。
- 没有显式对齐、预取或手写向量归约。

## 观察重点

如果 SIMD 版本比 baseline 有明显提升，说明 element-major 布局让内层工况循环获得了较好的向量化机会。

如果提升不明显，可以关注 `N` 是否过小、编译器是否生成向量 sqrt/max，以及内存带宽是否已经成为主要瓶颈。
