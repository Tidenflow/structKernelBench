# MaxStressEnvelope / CUDA / v1

## 当前实现

CUDA v1 使用“一单元一个 GPU 线程”的融合 kernel。每个线程负责一个单元，在寄存器中遍历该单元的所有工况并取最大值：

```cpp
int i = blockDim.x * blockIdx.x + threadIdx.x;
if (i >= M) return;

float max_vm = 0.0f;
for (int j = 0; j < N; ++j) {
    int idx = i * N + j;
    float ds = sx[idx] - sy[idx];
    float ss = sy[idx] - sz[idx];
    float ts = sz[idx] - sx[idx];
    float vm = sqrtf(0.5f * (ds*ds + ss*ss + ts*ts +
                              6.0f * (txy[idx]*txy[idx] + tyz[idx]*tyz[idx] + tzx[idx]*tzx[idx])));
    max_vm = fmaxf(max_vm, vm);
}
envelope[i] = max_vm;
```

源码位置：`MaxStressEnvelope/kernels/cuda_kernel.cu`

## 优化程度

这是 v1 的手写 CUDA 融合实现。它没有先生成所有工况的中间 Von Mises 数组，而是在一个 kernel 内完成“计算 Von Mises + 单元内 max”。

| 方向 | 当前状态 |
|---|---|
| 线程映射 | 一个 CUDA thread 负责一个单元 |
| block 大小 | `256` threads |
| 工况归约 | 单线程寄存器内串行 max |
| 中间数组 | 无，计算和归约融合 |
| 计时 | CUDA event 统计 kernel 与 transfer |

## 优化思路

这个版本的核心是融合：每个线程直接读取第 `i` 个单元的所有工况，边计算 Von Mises，边更新 `max_vm`。这样避免了把 `M*N` 个中间 Von Mises 值写回全局内存。

它的优点是：

- 中间结果留在寄存器中。
- 只输出 `M` 个 envelope 值。
- 代码路径和 baseline 语义很接近。
- 对每个单元的工况数据是连续读取。

## 性能瓶颈

当前映射简单，但不是所有场景都理想：

- `N` 很大时，一个线程要串行处理很多工况，线程内工作较重。
- `N` 很小时，每个线程工作量太少，GPU 吞吐可能不充分。
- 不同单元的线程同时读取连续段，访存合并效果受数据布局和 `N` 影响。
- 没有使用 warp 协作或 block 级归约来处理大 `N`。
- 每次调用仍重新分配、拷贝并释放 device memory。

## 观察重点

CUDA v1 适合观察“融合计算 + 寄存器归约”的手写 GPU 实现能达到什么吞吐。它和 CUB 的差异很有意义：CUDA v1 减少中间写入，CUB v1 则使用工业级分段归约原语。
