# Von Mises / Thrust / v1

## 当前实现

Thrust v1 使用 `thrust::transform` 作为工业级并行算法参考。六个输入数组通过 `zip_iterator` 组合成 tuple，`VonMisesFunctor` 对每个 tuple 计算一个输出：

```cpp
thrust::transform(
    thrust::device,
    thrust::make_zip_iterator(
        d_sx.begin(), d_sy.begin(), d_sz.begin(),
        d_txy.begin(), d_tyz.begin(), d_tzx.begin()),
    thrust::make_zip_iterator(
        d_sx.end(), d_sy.end(), d_sz.end(),
        d_txy.end(), d_tyz.end(), d_tzx.end()),
    d_vm.begin(),
    VonMisesFunctor());
```

源码位置：`VonMises/kernels/thrust_kernel.cu`

## 优化程度

这是当前项目中 Von Mises 的 GPU 库/算法参考实现。它不手写 CUDA kernel，而是把逐元素变换表达为 Thrust 并行算法。

| 方向 | 当前状态 |
|---|---|
| 核心算法 | `thrust::transform` |
| 输入组织 | `zip_iterator` 合并六个 device vector |
| 计算逻辑 | `VonMisesFunctor` |
| 内存管理 | `thrust::device_vector` 自动管理 |
| 对比意义 | 与手写 CUDA kernel 比较抽象层和性能差距 |

## 优化思路

Von Mises 计算本质上是一个 map 操作：

```text
(sx[i], sy[i], sz[i], txy[i], tyz[i], tzx[i]) -> vm[i]
```

Thrust v1 直接把这个 map 表达给库，由 Thrust 负责设备端并行调度。`zip_iterator` 避免手动构造结构体数组，同时保持六个 SoA 输入数组的接口。

## 与手写 CUDA 的区别

手写 CUDA v1 更透明，线程映射、block 大小、内存分配都在项目代码里。

Thrust v1 更偏声明式：

- 项目代码描述“对每个元素做 transform”。
- Thrust 管理 device vector。
- Thrust 生成并启动内部 kernel。
- 代码更简洁，但底层调度细节更少暴露。

## 性能观察重点

比较 CUDA 和 Thrust 时，可以重点观察：

- Thrust 的抽象开销是否明显。
- `device_vector` 构造和拷贝在端到端场景中的成本。
- kernel 时间是否接近手写 CUDA。
- 对于这种规整逐元素算子，库算法能否达到稳定吞吐。

如果 Thrust 和手写 CUDA 接近，说明当前算子足够规则，库抽象没有带来明显损失。如果差距较大，可能需要进一步区分 kernel 时间、传输时间和临时对象管理成本。
