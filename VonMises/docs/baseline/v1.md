# Von Mises / baseline / v1

## 当前实现

baseline 是 Von Mises 等效应力计算的单线程参考实现。输入采用 SoA 布局，每个应力分量是一条连续数组：

- `sx[i]`, `sy[i]`, `sz[i]`: 三个正应力分量。
- `txy[i]`, `tyz[i]`, `tzx[i]`: 三个剪应力分量。
- `vm[i]`: 第 `i` 个单元的 Von Mises 等效应力输出。

核心计算公式是：

```cpp
float ds = sx[i] - sy[i];
float ss = sy[i] - sz[i];
float ts = sz[i] - sx[i];
vm[i] = std::sqrt(0.5f * (ds*ds + ss*ss + ts*ts +
                           6.0f * (txy[i]*txy[i] + tyz[i]*tyz[i] + tzx[i]*tzx[i])));
```

源码位置：`VonMises/kernels/baseline.cpp`

## 优化程度

这是 v1 的正确性和性能基线。它不引入线程并行、显式 SIMD 或 GPU 调度，主要用于观察最直接逐元素计算的成本。

| 方向 | 当前状态 |
|---|---|
| 线程级并行 | 无，单线程顺序遍历 |
| 数据布局 | SoA，六个输入数组连续存储 |
| SIMD | 不显式使用 |
| GPU | 不使用 |
| 计时意义 | 作为所有优化后端的对照组 |

## 性能特征

和 CSR-SpMV 不同，Von Mises 计算没有间接访存，也没有跨元素依赖。每个 `i` 的计算完全独立，并且访问的是连续数组。

这个版本的耗时主要来自：

- 六个输入分量的连续读取。
- 多次乘加和一次平方根计算。
- 单线程无法利用多核或 GPU 并行。

## 为什么需要 baseline

baseline 保持公式最清晰，适合作为结果验证和性能参照。OpenMP、SIMD、CUDA、Thrust 的性能提升，都可以理解为在这个逐元素公式上增加不同层次的并行执行能力。
