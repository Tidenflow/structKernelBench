# Von Mises / CUDA / v1

## 当前实现

CUDA v1 使用“一元素一个 GPU 线程”的映射方式：

```cpp
int i = blockDim.x * blockIdx.x + threadIdx.x;
if (i >= N) return;

float ds = sx[i] - sy[i];
float ss = sy[i] - sz[i];
float ts = sz[i] - sx[i];
vm[i] = sqrtf(0.5f * (ds*ds + ss*ss + ts*ts +
                       6.0f * (txy[i]*txy[i] + tyz[i]*tyz[i] + tzx[i]*tzx[i])));
```

源码位置：`VonMises/kernels/cuda_kernel.cu`

## 优化程度

这是 v1 的手写 CUDA 实现。它保留 SoA 数据布局，将每个单元的等效应力计算交给一个 CUDA thread。

| 方向 | 当前状态 |
|---|---|
| 线程映射 | 一个 CUDA thread 负责一个元素 |
| block 大小 | `256` threads |
| 数据布局 | 六个输入数组 + 一个输出数组 |
| kernel 计时 | CUDA event 统计 kernel 时间 |
| transfer 计时 | 单独统计 H2D + D2H |

## 优化思路

Von Mises 是高度数据并行的逐元素计算。每个 `vm[i]` 只依赖同一个 `i` 的六个输入分量，没有线程间通信，也不需要共享内存归约。

CUDA v1 的 wrapper 做了这些工作：

- 为六个输入数组和一个输出数组分配 device memory。
- 将 `sx/sy/sz/txy/tyz/tzx` 从 host 拷贝到 device。
- 启动 `vonmises_cuda_kernel`。
- 将 `vm` 从 device 拷回 host。
- 用 CUDA event 分别记录传输时间和 kernel 时间。

## 性能特征

相比 CSR-SpMV 的 CUDA v1，Von Mises 更适合这种简单线程映射：

- 所有线程工作量几乎一致。
- 全部数组按 `i` 连续读取，访问模式规整。
- 不需要处理不同行长度和负载不均。
- 没有间接访存。

主要瓶颈通常来自全局内存带宽、`sqrtf` 吞吐和 host-device 数据传输。

## 当前限制

这个版本没有使用更激进的 GPU 优化：

- 没有持久化 device buffer。
- 每次 benchmark 调用都会重新分配和拷贝数据。
- 没有使用 pinned memory 或异步 stream。
- 没有把多个后处理步骤融合到同一个 kernel。

因此它适合观察“手写 GPU 逐元素 kernel”的基础性能，而不是完整后处理流水线的极限性能。
