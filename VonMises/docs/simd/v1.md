# Von Mises / SIMD / v1

## 当前实现

SIMD v1 使用和 baseline 相同的逐元素公式，但代码形态非常适合编译器自动向量化：

```cpp
for (int i = 0; i < N; ++i) {
    float ds = sx[i] - sy[i];
    float ss = sy[i] - sz[i];
    float ts = sz[i] - sx[i];
    vm[i] = std::sqrt(0.5f * (ds*ds + ss*ss + ts*ts +
                               6.0f * (txy[i]*txy[i] + tyz[i]*tyz[i] + tzx[i]*tzx[i])));
}
```

源码位置：`VonMises/kernels/simd.cpp`

## 优化程度

这是 v1 的自动向量化实现。当前代码没有手写 AVX/SSE intrinsic，而是依赖 Release 编译参数：

- `-O3`
- `-march=native`
- `-ffast-math`

| 方向 | 当前状态 |
|---|---|
| SIMD 方式 | 编译器自动向量化 |
| 显式 intrinsic | 无 |
| 数据布局 | SoA，连续数组 |
| 分支 | 无 |
| 适合目标 | 观察连续逐元素公式的向量化收益 |

## 优化思路

这个算子是自动 SIMD 的理想场景之一：

- 每个元素互相独立。
- 六个输入数组和一个输出数组都是连续访问。
- 循环内部没有分支。
- 没有 CSR 那种 `x[col_idx[k]]` 间接访存。
- 没有跨元素归约变量。

编译器可以把多个 `i` 打包到一个向量寄存器中，同时计算多个单元的 `ds`、`ss`、`ts`、剪应力平方和以及最终结果。

## 性能限制

当前 v1 仍然有几个现实限制：

- `sqrt` 的向量化和吞吐取决于编译器和 CPU 指令集。
- 读取六个输入数组会产生较高内存带宽需求。
- 没有显式控制对齐、预取或循环展开。
- 没有把 OpenMP 和 SIMD 显式组合成多线程向量化版本。

## 应该怎么看结果

如果 SIMD 比 baseline 明显更快，说明当前 SoA 布局和无分支公式让编译器成功生成了较好的向量代码。

如果提升有限，通常可以继续检查编译器优化报告、`sqrt` 是否被向量化，以及数据规模是否足够大到摊薄函数调用和计时噪声。
