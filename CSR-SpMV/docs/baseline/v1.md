# CSR-SpMV / baseline / v1

## 当前实现

baseline 是最朴素的 CSR-SpMV 实现。矩阵使用 CSR 三数组表示：

- `values[k]`: 第 `k` 个非零元的数值。
- `col_idx[k]`: 第 `k` 个非零元所在列。
- `row_ptr[i] ~ row_ptr[i + 1]`: 第 `i` 行非零元在 `values` 和 `col_idx` 中的范围。

核心计算是逐行遍历：

```cpp
for (int i = 0; i < N; ++i) {
    double sum = 0.0;
    int start = row_ptr[i];
    int end = row_ptr[i + 1];
    for (int k = start; k < end; ++k) {
        sum += values[k] * x[col_idx[k]];
    }
    y[i] = sum;
}
```

源码位置：`CSR-SpMV/kernels/baseline.cpp`

## 优化程度

这是 v1 的参考实现，优化程度刻意保持最低。它主要用于建立正确性基线和性能基线，不主动引入线程并行、SIMD 指令、数据重排或缓存分块。

| 方向 | 当前状态 |
|---|---|
| 线程级并行 | 无，单线程逐行计算 |
| 数据布局优化 | 使用标准 CSR，不做重排 |
| SIMD | 不显式使用 |
| GPU | 不使用 |
| 计时意义 | 作为所有并行版本的对照组 |

## 性能特征

CSR-SpMV 通常是内存带宽受限算子。每个非零元需要读取 `values[k]`、`col_idx[k]`，并通过 `col_idx[k]` 间接读取 `x`。真正的浮点计算只有一次乘法和一次加法，所以计算密度很低。

这个版本的耗时主要来自：

- `values` 和 `col_idx` 的线性读取。
- `x[col_idx[k]]` 的间接访存。
- 单线程无法利用多核 CPU。
- 每一行内部的归约依赖让指令级并行空间有限。

## 为什么仍然需要它

baseline 的价值不是快，而是稳定、清楚、可验证。其他实现的输出和吞吐量都应该和它对齐理解：

- OpenMP 观察的是“按行并行”带来的收益。
- SIMD 观察的是“编译器能否优化内层归约”。
- CUDA 观察的是“每行映射到 GPU 线程”后的吞吐变化。
- cuSPARSE 观察的是工业库对 CSR-SpMV 的综合优化上界。
